# 乐高零件管理系统 - 页面规划方案总结

---

## 一、页面架构设计

### 1.1 整体布局结构

系统采用 **三区域垂直布局**（AA-BB-CC模式），适配移动端和桌面端：

| 区域 | 位置 | 功能 | 样式特点 |
|------|------|------|----------|
| **AA区域** | 顶部 | Logo + 标题 | 固定高度，黄色背景（#F2CD37） |
| **BB区域** | 中部 | 导航标签页 | 横向排列，带背景图片的按钮 |
| **CC区域** | 底部 | 内容区域 | 白色卡片容器，自适应高度 |

### 1.2 标签页导航

四个核心功能模块通过标签页切换：

```
┌─────────────────────────────────────┐
│  [仓库管理]  [零件管理]  [零件搜索]  [系统设置]  │
└─────────────────────────────────────┘
```

- **仓库管理**（蓝色按钮）- 默认激活
- **零件管理**（橙色按钮）- 选择盒子后进入
- **零件搜索**（绿色按钮）- 全局搜索功能
- **系统设置**（橙色按钮）- 数据管理和统计

## 二、功能模块页面规划

### 2.1 仓库管理页面（repositories-tab）

**布局结构**：
```
┌─────────────────────────────────────┐
│  仓库列表          [添加仓库]       │
│  共 0 个仓库                         │
├─────────────────────────────────────┤
│ [仓库1] [仓库2] [仓库3] ...          │  ← 横向滚动
├─────────────────────────────────────┤
│  ▼ 仓库名 - 盒子管理                 │  ← 选中仓库后显示
│  [添加盒子] [盒子转仓]               │
│  共 0 个盒子                         │
├─────────────────────────────────────┤
│ ┌─────┐ ┌─────┐ ┌─────┐            │
│ │盒子1│ │盒子2│ │盒子3│ ...        │  ← 3列网格
│ │ 2P  │ │ 5P  │ │ 3P  │            │
│ └─────┘ └─────┘ └─────┘            │
└─────────────────────────────────────┘
```

**交互特性**：
- 点击仓库卡片 → 选中并加载盒子列表
- 长按仓库卡片 → 编辑仓库名称（内联编辑）
- 盒子卡片显示零件数量（X P）

- 零件图片来自 Rebrickable API

## 三、模态框组件规划

系统设计了多个模态框组件，统一使用遮罩层 + 卡片布局：

| 模态框名称 | 功能 | 触发方式 |
|------------|------|----------|
| **添加零件** | 表单填写零件信息 | 点击"添加零件"按钮 |
| **零件选择器** | 搜索并选择零件 | 点击"选择零件"按钮 |
| **颜色选择器** | 搜索并选择颜色 | 点击"选择颜色"按钮 |
| **数量编辑** | 快速增减数量 | 长按零件卡片 |
| **零件详情** | 查看零件完整信息 | 点击零件卡片 |
| **CSV导入** | 批量导入零件数据 | 点击"批量导入"按钮 |

## 四、UI设计规范

### 4.1 色彩系统

| 颜色变量 | 值 | 用途 |
|----------|-----|------|
| `--primary-color` | #F2CD37 | 页面背景（乐高黄色） |
| `--secondary-color` | #2196F3 | 主按钮、选中状态 |
| `--success-color` | #4CAF50 | 成功状态、新品标签 |
| `--warning-color` | #FF9800 | 警告状态、旧品标签 |
| `--danger-color` | #F44336 | 删除按钮、数量高亮 |

### 4.2 响应式设计

系统支持移动端和桌面端自适应：

- **移动端**（<768px）：3列盒子网格，卡片宽度 90px
- **桌面端**（≥768px）：4列盒子网格，最大宽度 1200px，居中显示

### 4.3 交互设计规范

| 交互类型 | 实现方式 | 适用场景 |
|----------|----------|----------|
| **点击** | `addEventListener('click')` | 导航、选择、按钮操作 |
| **长按** | 800ms 定时器（mousedown/touchstart） | 编辑名称、快速编辑数量 |
| **滑动** | 横向滚动容器 | 仓库列表、盒子网格 |
| **表单验证** | 实时错误提示 | 添加零件、批量导入 |
| **状态反馈** | CSS类切换（active/selected） | 标签页切换、选中状态 |

## 六、页面状态管理

系统使用简单的全局变量管理页面状态：

```javascript
let selectedRepository = null;  // 当前选中的仓库
let selectedBox = null;         // 当前选中的盒子
let editingRepository = null;   // 正在编辑的仓库
let editingBox = null;          // 正在编辑的盒子
```

## 七、页面规划总结

| 维度 | 规划要点 |
|------|----------|
| **架构** | 三区域垂直布局，标签页切换模式 |
| **导航** | 四个核心功能模块，带图标背景的导航按钮 |
| **交互** | 点击选择 + 长按编辑，模态框集中处理表单 |
| **响应式** | 移动端优先，自动适配桌面端 |
| **视觉** | 乐高黄色主题，统一的卡片设计风格 |
| **PWA** | 离线缓存、本地存储、全局错误处理 |
| **扩展性** | 支持转仓、转盒等预留功能按钮 |

当前页面规划已完成核心功能实现，预留了转仓和转盒功能的入口，后续可根据需求扩展。
